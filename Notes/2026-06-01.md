# Node.js Backend Notes - 2026-06-01

Today I cleaned up the backend structure of my journal app and learned a few important Express concepts.

## What Changed

Earlier, route files were doing too many things:

- Defining API paths
- Reading request data
- Calling services
- Sending responses
- Handling errors

Now the backend flow is cleaner:

```txt
request
  -> middleware
  -> route
  -> controller
  -> service
  -> JSON file / database
```

## Route, Controller, Service

I learned that these three should have separate jobs:

```txt
Route = decides which URL calls which function
Controller = handles req and res
Service = contains actual app logic
```

Example:

```js
router.post('/login', authController.login)
```

The route is now simple. The controller handles the request and response. The service checks the data.

## Middleware

Middleware is just a function that Express runs during the request flow.

Examples from this project:

```js
app.use(cors({ origin: 'http://localhost:5173' }))
app.use(express.json())
```

`express.json()` is important because it allows me to use:

```js
req.body
```

Middleware runs from top to bottom. If it calls `next()`, the request moves forward. If it sends a response, the request usually stops there.

## Global Error Handler

Before, every controller had its own `try/catch`.

Now errors are handled in one place using:

```js
throw new AppError('Journal not found', 404)
```

Then the global error handler sends the response:

```js
res.status(statusCode).json({ error: message })
```

This keeps controllers cleaner.

## app.js and server.js

I also split `server.js` into two files:

```txt
app.js = creates and configures the Express app
server.js = starts the server
```

This makes the project structure easier to understand and easier to expand later.

## Main Takeaway

Today’s biggest lesson:

```txt
Keep each file responsible for one clear job.
```

Routes should stay small, controllers should handle request/response, services should hold logic, and errors should go to one common error handler.
